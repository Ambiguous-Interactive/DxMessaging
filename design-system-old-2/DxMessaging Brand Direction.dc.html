A messaging system, and a way to think about how a game fits together. Model each interaction as a typed message, and your systems stop holding references to each other.

Reading the brief
The mark should be the product. DxMessaging routes typed messages between systems that don't know each other. So the symbol is a node on a bus: a hexagon with a live signal passing through it.
Why the hexagon. It reads as one cell in a network, tiles into patterns, and shares geometric DNA with the studio world without copying anyone.
Electric violet → signal cyan. Violet gives it an engineered, premium feel. Cyan marks the live message moving through. Near-black backgrounds let both colors carry the work.
Friendly by default. Your docs voice is already warm, so the type stays a clean grotesk. A real upgrade from Roboto that still sounds like you.

05  Iconography  · the three primitives you think in
Three primitives. That's the whole paradigm.
You don't build your game into a framework. You pick which of three messages fits the moment, and your systems stay decoupled by default. Each one maps to a way people already communicate.
Geometry from
the Router mark
Untargeted
= the PA system
A global announcement with no sender and no recipient. Anything listening for it will hear it. Reach for it when the whole app needs to know.
[DxUntargetedMessage]
e.g.  GamePaused · SettingsChanged · SceneLoaded
Symmetric on purpose: no source, no target, everyone hears it.
Targeted
= the addressed letter
A command to one specific entity. Only that GameObject receives it. Directed, intentional.
[DxTargetedMessage]
e.g.  Heal(player) · Open(door) · Damage(enemy)
Broadcast
= the radio station
A fact emitted by a specific source. No intended recipient, but listeners can filter by who sent it. Good for UI, analytics, and achievements.
[DxBroadcastMessage]
e.g.  TookDamage · ItemPickedUp · ChestOpened
The ringed source is the tell: waves leave a known origin.

Unity · Type-safe messaging
Decouple your Unity systems with typed messages.
DxMessaging replaces tangled C# events and UnityEvents. Define a message, send it from anywhere, and any system can react without a reference to the sender.
Get started → ★  Star on GitHub
$ openupm add com.wallstop-studios.dxmessaging ⧉
Heal.cs
[DxTargetedMessage]
public partial struct Heal { int amount; }
// listen, cleanup is automatic
Token.RegisterGameObjectTargeted<Heal>(OnHeal);
// send from anywhere
new Heal(50).EmitGameObjectTargeted(player);

Message Types

DxMessaging gives you three message types. Choosing between them is the whole skill, and each one maps onto a way people already communicate.
◆ Note
If it matters who sent it, reach for Broadcast or Targeted, never Untargeted.

Define a message

Mark a struct with an attribute. The source generator does the rest.
csharp ⧉ copy
[DxBroadcastMessage]
[DxAutoConstructor]
public readonly partial struct TookDamage {
  public readonly int amount;
}
✓ Tip
Pass messages by ref in handlers to keep dispatch zero-allocation on hot paths.
